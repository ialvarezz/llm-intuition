What is an LLM

A large language model does exactly one thing: given a sequence of tokens, it predicts what comes next. Everything you've seen an LLM do is that single operation, repeated.

The only operation

An LLM is a next-token prediction machine. Given a sequence of tokens, it outputs a probability distribution over what token comes next — that's the whole operation. Chat, code generation, reasoning, summarization: everything you've watched an LLM do is that single step, called over and over, with the growing output fed back in as new input. A model pretrained on enough text implicitly picks up grammar, facts, and reasoning patterns because predicting the next token correctly requires something that behaves like understanding them — none of it is programmed in directly. A raw base model pretrained this way is still just a text continuer; it takes a further stage, fine-tuning on human-written example responses, to turn it into something that behaves like an assistant rather than an autocomplete (see note 07).

The generation loop

Prompt → tokenize → forward pass → distribution → sample one token → append → repeat, until a stop token or a length limit is hit. That's the identical loop behind every kind of output. The model itself is stateless between calls — it has no internal memory of the previous step. Each forward pass reprocesses the entire sequence so far from scratch; the growing sequence of tokens is the state, and nothing else. That's also why context windows are a hard limit rather than a setting (see note 05).

What it is not

Not a knowledge base with lookups — there's no query engine matching your question to a stored record; every output is a generated continuation shaped by statistical patterns, not retrieved from an index. Not deterministic by default — sampling is inherent to the loop, and even at the most conservative settings identical inputs can produce different outputs. It doesn't "read" text the way you do; it consumes tokens, not words or characters (see note 02). And it has no persistent memory between conversations — a new session starts with an empty sequence.

Pro vs. amateur

Pros think in tokens and distributions, not words and answers — they expect re-running the same prompt to change the output, because "the answer" was always one sample from a chain of distributions. Pros know hallucination is the mechanism working as designed: the model was trained to produce a fluent, statistically likely continuation, not a correct one — fluent and true are two different properties that happen to coincide often, not the same thing. Pros know the model can't reliably count its own output, because it only ever sees tokens, not the underlying characters or words. And pros know "the model learned X" really means "X was statistically present in the training data" — nothing more, nothing verified.
