The 15-Minute Primer

You don't need a PhD to build AI plugins. You need four ideas: tokens, context, tool calls, and the loop. Here they are — everything else in this module builds on them.

ONE TASK · FOUR HOPS
Every agent you'll meet in this module — and every plugin — lives somewhere on this loop.

Next-token prediction, looped

An LLM predicts the next token — that's the whole operation (see note 01). Chat is that same operation run in a loop: your message becomes tokens, the model predicts a continuation, and your next message extends the sequence and starts the cycle again. The context window is the model's entire working memory — everything it knows about the current conversation has to fit inside it (see note 05). That matters here because every plugin, tool, or piece of injected context competes for space in that same window.

Tools: the model asks, the harness acts

A raw LLM can't do anything except emit tokens — it can't run a command or call an API. Tool use lets the model return a structured call describing an action; the surrounding application executes it and sends the result back. The model's job stops at "returns a structured call" — it doesn't touch a filesystem or a network socket. It's also not compelled to call a tool on every turn: it decides, based on the request and the tool's description, whether to call a tool or respond directly.

An agent is a loop

Chain that decision together and you get an agent: the model decides, the surrounding program acts, the result comes back into the context window as more tokens, and the cycle repeats until the model decides it's done. A workflow is the same pieces wired along a path a human coded in advance; an agent is those same pieces with the model itself choosing what happens next, turn by turn. Mechanically nothing new is happening — it's the same next-token-prediction loop as ordinary chat, extended so some of each turn's input is tool output instead of a human typing.

Plugins extend the harness

The model is fixed — its weights don't change between one tool call and the next. What varies from one agentic coding tool to another is the program wrapped around the model: the thing that reads the model's tool-call output, decides which functions are available, executes them, and feeds the result back in. Call that surrounding program the harness — Claude Code and GitHub Copilot are harnesses in this sense. A plugin is not a modification to the model. It's a modification to the harness: a new tool the harness exposes, a new piece of context it injects, a new command it recognizes. That framing is this module's thesis.

Pro vs. amateur

The model never executes anything — every "action" you see an agent take is the harness honoring a structured request the model emitted. Agent quality is mostly a function of harness and context quality, not raw model IQ: the same model looks brilliant or useless depending on what tools it's given and how well they're described. And token costs apply to everything a plugin injects — tool descriptions, retrieved context, and instructions all sit inside the same bounded context window and get billed as input tokens on every turn, whether or not the model ends up using them.
